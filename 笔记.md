# Spring Boot 拦截器【核心笔记・极简回顾版】
一、核心作用
在Controller 执行前做校验：登录验证、权限拦截、请求预处理。
二、配置固定 2 步
1. 写拦截器
   实现 HandlerInterceptor 接口，重写 preHandle（90% 场景只用这个）  
`   public class LoginInterceptor implements HandlerInterceptor {
   // 前置拦截：Controller执行前调用  
   @Override  
   public boolean preHandle(...) {  
   // 校验逻辑：未登录则拦截  
   return true; // true=放行 | false=拦截  
   }  
   }`
2. 注册拦截器
   @Configuration + WebMvcConfigurer 注册并配置路径  
`   @Configuration  
   public class MvcConfig implements WebMvcConfigurer {  
   @Override  
   public void addInterceptors(InterceptorRegistry registry) {  
   registry.addInterceptor(new LoginInterceptor())  
   .addPathPatterns("/**")        // 拦截所有  
   .excludePathPatterns(          // 放行接口  
   "/user/login", "/user/code"  
   );  
   }  
   }`
   三、关键规则  
   /**：拦截所有接口  
   /user/**：拦截 /user/ 下所有接口  
   preHandle return true → 放行进 Controller  
   preHandle return false → 直接拦截，不进 Controller  
   四、请求执行流程（必记）  
   浏览器请求 → DispatcherServlet（总入口） → 拦截器 preHandle → Controller  
   五、核心特点  
   拦截器是 Spring MVC 组件，只拦截 Controller 请求  
   由框架自动调用，无需手动触发  

# 登录校验拦截器 LoginInterceptor
1. 作用
   统一拦截需要登录的接口，校验用户登录状态，未登录直接拒绝。
2. 拦截规则
   拦截：/** 所有接口
   放行：/user/login、/user/code、/user/logout（无需登录）
3. 核心逻辑
   检查 Session 中是否有登录用户
   无用户 → 拦截，返回未登录
   有用户 → 存入UserHolder，放行到 Controller
   请求结束 → 清空用户上下文
4. 核心价值
   统一校验、保护接口、代码简洁、全局获取用户
